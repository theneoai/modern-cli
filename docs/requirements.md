# modern-ai-cli 功能需求分析

> 本文档梳理 modern-ai-cli 项目的功能需求，涵盖用户画像、核心功能、非功能需求及使用场景。

---

## 一、用户画像

| 用户类型 | 描述 | 使用场景 |
|---------|------|---------|
| **开发者** | 软件工程师、架构师 | 代码生成、调试、重构、API 设计 |
| **DevOps/SRE** | 运维工程师 | Shell 脚本编写、系统诊断、自动化 |
| **技术写作者** | 文档工程师 | 技术文档撰写、代码注释 |
| **AI 爱好者** | 普通终端用户 | 日常问答、学习新知、创意写作 |
| **团队负责人** | 技术管理者 | 多 Agent 协作任务、代码审查 |

---

## 二、核心功能需求

### 2.1 基础 AI 对话（`ai ask`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 单次问答 | 快速提问，即时获得答案 | P0 |
| 多模式提示 | 7 种模式（default/code/explain/refactor/debug/creative/shell） | P0 |
| 流式输出 | 实时显示 AI 回复内容 | P0 |
| Token 统计 | 显示输入/输出 Token 数量 | P1 |
| 自定义系统提示 | 通过 `--system` 覆盖默认提示 | P1 |
| 非流式模式 | `--no-stream` 完整输出 | P2 |

### 2.2 交互式聊天（`ai chat`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 持久化会话 | 创建、保存、加载会话 | P0 |
| 斜杠命令 | `/clear`, `/mode`, `/save`, `/load`, `/history`, `/usage`, `/model`, `/help`, `/exit` | P0 |
| 上下文管理 | 自动裁剪历史消息（可配置长度） | P0 |
| 自动保存 | 每 5 轮对话自动保存 | P1 |
| 会话恢复 | 通过 `--session` 参数恢复指定会话 | P1 |
| 模式切换 | 会话中实时切换 AI 模式 | P1 |

### 2.3 代码生成（`ai generate`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 类型选择 | component / function / test / docs / script | P0 |
| 语言选择 | TypeScript/JavaScript/Python 等 | P0 |
| 交互式向导 | 通过 @clack/prompts 引导用户 | P0 |
| 完整输出 | 包含类型定义、错误处理、注释、示例 | P1 |

### 2.4 配置管理（`ai config`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 配置持久化 | JSON 格式本地存储 | P0 |
| 配置向导 | `wizard` 子命令引导首次设置 | P0 |
| 模型选择 | claude-opus-4-6 / sonnet-4-6 / haiku-4-5 | P0 |
| 主题切换 | dark / light | P1 |
| 历史设置 | 启用/禁用历史、最大消息数 | P1 |
| 流式设置 | 启用/禁用流式输出 | P1 |
| 配置路径 | 显示配置文件位置 | P2 |

### 2.5 MCP 技能系统（`ai mcp`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 技能管理 | list / enable / disable / install / remove | P0 |
| 内置技能 | shell / files / http / calculator | P0 |
| 远程安装 | 从 npm 安装技能包 | P1 |
| 本地安装 | 安装本地 .mjs 文件 | P1 |
| 技能搜索 | 搜索远程技能注册表 | P1 |
| 工具调用 | 直接调用技能工具（调试/脚本） | P2 |

### 2.6 多 Agent 系统（`ai agent`）

#### 2.6.1 基础编排

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 自动规划 | LLM 分解任务并执行 | P0 |
| 内置角色 | researcher / planner / coder / reviewer / synthesizer | P0 |
| 并行/顺序执行 | 支持两种执行模式 | P0 |
| 可视化看板 | AgentBoard 实时显示执行状态 | P1 |
| 单 Agent 执行 | `solo` 子命令运行单个 Agent | P1 |

#### 2.6.2 自定义 Agent

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 创建自定义 Agent | 定义名称、描述、图标、标签、系统提示 | P1 |
| 编辑 Agent | 修改自定义 Agent 属性 | P1 |
| 查看 Agent | 显示完整元数据和系统提示 | P1 |
| 删除 Agent | 移除自定义 Agent | P1 |
| 导入/导出 | JSON 格式共享 Agent 配置 | P2 |

#### 2.6.3 工作流管理

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 工作流定义 | JSON 格式定义多步骤流程 | P1 |
| 工作流执行 | `run` 执行保存的工作流 | P1 |
| 工作流模板 | 内置模板快速创建 | P2 |
| 工作流管理 | list / save / remove | P1 |

#### 2.6.4 记忆系统

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 记忆类型 | episodic / semantic / procedural / working | P1 |
| 添加记忆 | 手动添加记忆条目 | P1 |
| 搜索记忆 | 关键词搜索 | P1 |
| 记忆蒸馏 | AI 压缩和总结旧记忆 | P2 |
| 共享记忆池 | 所有 Agent 可访问的共享记忆 | P1 |
| 记忆统计 | 跨 Agent 记忆统计 | P2 |

#### 2.6.5 会议系统

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 多 Agent 会议 | 多 Agent 结构化讨论 | P1 |
| 会议模式 | roundtable / debate / brainstorm / review | P1 |
| 多轮对话 | 可配置讨论轮数 | P1 |
| 会议记录 | 保存完整会议记录 | P1 |
| 会议看板 | MeetingBoard 可视化会议进程 | P2 |

#### 2.6.6 组织架构

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 组织结构 | 公司/部门层级管理 Agent | P2 |
| 组织运行 | 按公司/部门批量运行 Agent | P2 |
| 组织图表 | 可视化组织架构 | P2 |

### 2.7 插件系统（`ai plugin`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 插件加载 | 动态加载用户安装插件 | P1 |
| 插件命令 | install / remove / list / enable / disable | P1 |

### 2.8 版本与更新（`ai whatsnew`）

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 更新日志 | 查看版本变更 | P1 |
| 构建信息 | 显示 git hash、构建时间 | P1 |
| 升级通知 | 后台检查新版本（24h 间隔） | P1 |

---

## 三、非功能需求

| 需求项 | 描述 |
|--------|------|
| 跨平台 | 支持 Linux x64/arm64、macOS x64/arm64、Windows x64 |
| Node.js 版本 | 最低 Node.js 18+ |
| 配置验证 | JSON Schema 验证配置合法性 |
| 错误处理 | 优雅的错误提示和退出码 |
| 主题系统 | 统一的 UI 主题和图标 |
| Markdown 渲染 | 终端友好的 Markdown 显示 |
| 自适应思考 | Claude Opus 4-6 支持 reasoning 模式 |

## 五、使用场景示例

| 场景 | 命令 |
|------|------|
| 快速提问 | `ai ask "解释 Promise"` |
| 写代码 | `ai ask --mode code "写个防抖函数"` |
| 交互对话 | `ai chat` |
| 生成组件 | `ai gen --type component "Modal对话框"` |
| 多 Agent 任务 | `ai agent run "构建 REST API 并写测试"` |
| 启用技能 | `ai mcp enable shell` |
| 查看记忆 | `ai agent memory list` |
| 组织会议 | `ai agent meet start "讨论架构设计"` |
